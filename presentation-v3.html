DR Flip Sequence

Step-by-step failover from us-east-1 to us-west-2 during a sustained regional outage
06 / 07
Failover Steps
1
Assess & Approve
Confirm sustained us-east-1 outage via AWS Health Dashboard. Check S3 CRR replication status. Get VP Engineering approval.
5 minutes
2
Activate Platform APIs
Route53 failover → west2 ALB. DynamoDB west2 becomes write region. Scale up ECS services. Verify Lambda functions.
5-10 minutes
3
Activate Iceberg Tables
Validate latest metadata.json in west2 S3. Run register_table in west2 Glue. Verify referenced data files exist (staleness buffer).
10-15 minutes
4
Activate Compute Platforms
Snowflake: Run external table recreation automation. Starburst: Scale up west2 cluster. Databricks: Activate workspace. Immuta: Verify connectivity.
10-20 minutes (parallel)
5
Activate Notifications
Verify SNS west2 receiving messages. Confirm Kafka flowing via west2 DX. Verify Polling API reachable via CNAME.
5 minutes (parallel with Step 4)
6
Producers Flip
Reverse CRR (west2→east1). Switch writes to west2 S3. Non-flipped: no change (Private Listing continues).
Coordinated with producers
7
Notify Consumers & Monitor
Status: Platform operational in west2. List available vs unavailable datasets. Continuous monitoring.
5 minutes
